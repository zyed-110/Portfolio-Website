# Zyed A | AI & Software Engineering Portfolio

A modern, responsive, and highly technical portfolio website built to showcase projects in Machine Learning, Computer Vision, Backend Development, and Data Science. 

Designed with a focus on deep technical immersion, featuring dynamic dashboard interfaces, animated data visualizations, and robust project architectures.

## 🚀 Features

- **Interactive Vibe-Coding Dashboard**: View engineered projects through a custom terminal-styled UI with smooth Anime.js transitions.
- **Dynamic Technology Stack Section**: Organized categories for Programming Languages, Frontend, Backend, ML, and Developer Tools.
- **Interactive Certificates Hub**: Preview or download certifications directly via an embedded modal viewer.
- **Custom Hardware Overlays**: Unique visual identity with glowing cursors, particle backgrounds, and glassmorphism elements.
- **Smart Contact System**: Seamless integration for Desktop (Email) and Mobile (WhatsApp/Email).
- **Fully Responsive**: Optimized fluid design accommodating mobile, tablet, and widescreen aspect ratios without overflow.

## 💻 Tech Stack

- **Frontend core**: HTML5, CSS3 (Vanilla), JavaScript (ES6+)
- **Animation Frameworks**: Anime.js
- **Icons**: FontAwesome 6
- **Hosting/Deployment**: GitHub Pages

## 📂 Folder Structure

```
Portfolio/
├── assets/
│   ├── css/          # Custom stylesheets (style.css)
│   ├── js/           # Interactive logic and JSON datasets (script.js)
│   ├── images/       # Static branding and background assets
│   ├── screenshots/  # Project demo images and videos
│   ├── certificates/ # PDF documents for certifications
│   └── resume/       # Current Resume PDF
├── index.html        # Main entry point 
├── reorganize.bat    # Script to organize raw files into assets directory
└── README.md
```

## 🛠️ Installation & Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/zyed-110/Portfolio-Website.git
   ```
2. **Reorganize Assets (If necessary):**
   If files are in the root directory, simply run `reorganize.bat` (Windows) to instantly map the repository to the professional `assets/` structure.
3. **Run Locally:**
   Simply open `index.html` in any modern web browser. No local server required, though using VS Code Live Server is recommended for active development.

## 🌐 Deployment

This project is configured for seamless deployment via **GitHub Pages**. 
Simply push to the `main` branch, and ensure the Pages setting is tracking the root folder.
